       ```typescript

         redirectUri: string;
         baseUrl: string;
       }
       ```

    2. **getOnshapeToken(userId: string): Promise<string>**
       - Fetches credentials from D1 for user
       - Checks if token expires within 5 minutes
       - Refreshes if needed using refreshOnshapeToken()
       - Returns valid access token
       - Throws ApiError if no credentials exist

    3. **refreshOnshapeToken(userId: string, refreshToken: string): Promise<TokenResponse>**
       - POST to https://cad.onshape.com/oauth/oauth/token
       - Body: grant_type=refresh_token, refresh_token={token}, client_id={id}, client_secret={secret}
       - Returns { access_token, refresh_token, expires_in }
       - Updates D1 with new tokens and expiry
       - Implements retry with exponential backoff (3 attempts)

    4. **storeOnshapeTokens(userId: string, tokens: TokenResponse): Promise<void>**
       - Upserts credentials in onshape_credentials table
       - Calculates expires_at from current time + expires_in
       - Sets created_at and last_used_at timestamps

    5. **generateAuthUrl(state: string): string>**
       - Returns Onshape OAuth authorization URL
       - Parameters: response_type=code, client_id, redirect_uri, scope, state
       - Scope: Request read access to documents and assemblies

    Follow existing patterns from gcalSync.ts:
    - Use shared/errors/api.ts for ApiError
    - Implement exponential backoff for retries
    - Log errors to console for debugging

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Onshape OAuth | User authorization flow with state parameter validation |
| Cloudflare Workers → Onshape Token Endpoint | Token exchange with client credentials |
| D1 → Token Storage | Per-user OAuth tokens with expiry tracking |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-01 | S | Client Secret exposure | mitigate | Stored as Cloudflare Worker secret, never logged or exposed to client |
| T-78-02 | T | OAuth token in transit | mitigate | HTTPS enforced for all Onshape API calls |
| T-78-03 | I | State parameter manipulation | mitigate | Generate secure random state, validate on callback, short TTL |
| T-78-04 | I | Authorization code interception | mitigate | Use PKCE if Onshape supports it, validate redirect_uri |
| T-78-05 | D | Token leakage via CORS/XSS | mitigate | All API calls proxy through backend, no tokens in client storage |
| T-78-06 | E | Token refresh failure | accept | Implement retry logic, user can re-authorize if refresh fails |
| T-78-07 | R | Excessive OAuth scopes | mitigate | Request minimum required scopes (read-only for documents/assemblies) |
</threat_model>
