    1. **GET /api/onshape/export/stl/:documentId/:elementId**
       - Requires cf-access-authenticated-user-email (Zero Trust)
       - Query params:
         - units: "millimeter" (default) or "meter"
         - mode: "binary" (default) or "ascii"
       - Calls Onshape endpoint: GET /api/partstudios/{eid}/stl
       - Streams response directly to client
       - Headers:
         - Content-Type: application/sla
         - Content-Disposition: attachment; filename="{documentName}_{elementName}.stl"
       - Handles errors from Onshape API

    Implementation details:
    ```typescript
    app.get('/export/stl/:documentId/:elementId', typedHandler({
      query: z.object({
        units: z.enum(['millimeter', 'meter']).default('millimeter'),
        mode: z.enum(['binary', 'ascii']).default('binary'),
      }),
      params: z.object({
        documentId: z.string().length(24),
        elementId: z.string().length(24),
      }),
    }, async (c, { params, query }) => {
      const userId = c.get('cf-access-authenticated-user-email');
      const token = await getOnshapeToken(userId);

      const response = await fetch(
        `${ONSHAPE_BASE_URL}/api/partstudios/${params.elementId}/stl?` +
        `units=${query.units}&mode=${query.mode}`,
        {
          headers: { Authorization: `Bearer ${token}` }
        }
      );

      if (!response.ok) throw new ApiError(response.status, 'Export failed');

      // Stream response to client
      return new Response(response.body, {
        headers: {
          'Content-Type': 'application/sla',
          'Content-Disposition': `attachment; filename="${params.documentId}_${params.elementId}.stl"`,
        }
      });
    }));
    ```

    Update main router:

    ```typescript
    app.route('/export', exports);
    ```

    1. **POST /api/onshape/export/step/:documentId/:elementId**
       - Requires cf-access-authenticated-user-email
       - Body: { format: "step" }
       - Calls Onshape: POST /api/partstudios/{eid}/export
       - Returns: { exportId, status: "pending"|"processing"|"done" }

    2. **GET /api/onshape/export/status/:exportId**
       - Requires cf-access-authenticated-user-email
       - Calls Onshape: GET /api/translations/{exportId}
       - Returns: { exportId, status, resultUrl?, failureReason? }
       - Stores export state in D1 or Workers KV (TTL: 1 hour)

    3. **GET /api/onshape/export/download/:exportId**
       - Requires cf-access-authenticated-user-email
       - Proxies download from Onshape resultUrl
       - Streams file to client
       - Returns Content-Type: application/octet-stream

    Implementation details:
    ```typescript
    // STEP export initiation
    app.post('/export/step/:documentId/:elementId', typedHandler({
      params: z.object({
        documentId: z.string().length(24),
        elementId: z.string().length(24),
      }),
      body: z.object({
        format: z.literal('step'),
      }),
    }, async (c, { params }) => {
      const userId = c.get('cf-access-authenticated-user-email');
      const token = await getOnshapeToken(userId);

      const response = await fetch(
        `${ONSHAPE_BASE_URL}/api/partstudios/${params.elementId}/export`,
        {
          method: 'POST',
          headers: {
            Authorization: `Bearer ${token}`,
            'Content-Type': 'application/json',
          },
          body: JSON.stringify({
            formatName: 'STEP',
            destinationTypeId: exportIds,
          }),
        }
      );

      const data = await response.json();
      const exportId = data.id;

      // Store export state
      await env.ONSHAPE_EXPORTS.put(exportId, JSON.stringify({
        userId,
        documentId: params.documentId,
        elementId: params.elementId,
        status: 'pending',
        createdAt: Date.now(),
      }), { expirationTtl: 3600 });

      return c.json({ exportId, status: 'pending' });
    }));
    ```

    Use Workers KV for export state storage (faster than D1 for polling)
