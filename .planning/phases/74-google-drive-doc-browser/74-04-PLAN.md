## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Dashboard calls protected /files endpoint |
| UI → External | Document links open google.com in new tab |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-14 | T | Unauthenticated dashboard access | mitigate | Dashboard route requires auth via TanStack Router, /files endpoint requires ensureAdmin |
| T-74-15 | I | Search query injection | mitigate | Search parameter passed through Zod schema on backend, no raw SQL (Drive API query parameter) |
| T-74-16 | R | URL manipulation | accept | Opening external URLs is read-only, noopener noreferrer prevents tab nabbing |
| T-74-17 | X | External link phishing | accept | Links go to official Google domains (docs.google.com), user expects external navigation |

## Security Checklist (Zero Trust SKILL.md)
- [ ] No decision based on Referer/Host headers
- [ ] Dashboard uses TanStack Router auth (session required)
- [ ] API endpoint has ensureAdmin middleware
- [ ] External links use rel="noopener noreferrer"
- [ ] No sensitive data in client-side logs
</threat_model>
