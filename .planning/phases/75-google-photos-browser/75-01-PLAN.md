## Photos API Response Format (from CONTEXT.md)

```typescript
// Google Photos mediaItems:list API response structure
interface MediaItem {
  id: string;
  filename: string;
  mimeType: string;  // image/jpeg, image/png, etc. (NOT video/* per PHOTO-02/D-01)
  baseUrl: string;   // Use for thumbnail (200x200px per D-03/D-04)
  mediaMetadata: {
    width: string;
    height: string;
    creationTime: string;  // ISO 8601 timestamp
  };
  description?: string;
}

// Google Photos albums:list API response structure
interface Album {
  id: string;
  title: string;
  mediaItemsCount?: string;
  coverPhotoBaseUrl?: string;
}
```

## Key Decisions from CONTEXT.md

- **D-01:** Filter server-side by mimeType — include only image types, exclude videos (video/mp4, video/quicktime, video/x-msvideo)
- **D-03/D-04:** Use Photos API baseUrl for thumbnails at 200x200px
- **D-05/D-06:** Display metadata (filename, dimensions, creation time, description)
- **D-06:** Page size 25 with pageToken pagination
- **D-08/D-09:** Albums with cover image and title
- **D-10:** Click album to filter media items by albumId
- **D-11/D-12/D-13:** Upload with multiple files, title, description, album select

    3. Define photoMediaItemSchema with z.object containing:
       - id: z.string().openapi({ description: "Google Photos media item ID" })
       - filename: z.string().openapi({ description: "Original filename" })
       - mimeType: z.string().openapi({ description: "MIME type (images only, no videos per PHOTO-02)" })
       - baseUrl: z.string().url().openapi({ description: "Base URL for photo access (use =w200-h200 for thumbnails per D-04)" })
       - width: z.string().optional().openapi({ description: "Image width in pixels" })
       - height: z.string().optional().openapi({ description: "Image height in pixels" })
       - creationTime: z.string().datetime().optional().openapi({ description: "Photo creation time (ISO 8601)" })
       - description: z.string().optional().openapi({ description: "User-provided description" })

    4. Define photoAlbumSchema with z.object containing:
       - id: z.string().openapi({ description: "Album ID" })
       - title: z.string().openapi({ description: "Album title" })
       - mediaItemsCount: z.string().optional().openapi({ description: "Number of items in album" })
       - coverPhotoBaseUrl: z.string().url().optional().openapi({ description: "Cover photo URL (use =w200-h200 for thumbnail)" })

    5. Define listMediaQuerySchema with z.object containing:
       - albumId: z.string().optional().openapi({ description: "Filter by album ID per D-10" })
       - pageToken: z.string().optional().openapi({ description: "Pagination token from Photos API" })
       - pageSize: z.coerce.number().min(1).max(50).default(25).optional().openapi({ description: "Items per page (default 25 per D-06)" })

    6. Define listMediaResponseSchema with z.object containing:
       - mediaItems: z.array(photoMediaItemSchema)
       - nextPageToken: z.string().optional()

    7. Define listAlbumsResponseSchema with z.object containing:
       - albums: z.array(photoAlbumSchema)
       - nextPageToken: z.string().optional()

    8. Define uploadPhotosRequestSchema with z.object containing:
       - Note: This endpoint uses multipart/form-data, handled differently in OpenAPI
       - The contract will reference the form data structure
       - Files: Array of image files (JPG, PNG, WEBP per D-11)
       - title: z.string().optional() - Title for all uploaded photos
       - description: z.string().optional() - Description for all uploaded photos
       - albumId: z.string().optional() - Target album ID per D-12

    9. Define uploadPhotosResponseSchema with z.object containing:
       - uploadedCount: z.number().openapi({ description: "Number of photos successfully uploaded" })
       - failures: z.array(z.object({
           filename: z.string(),
           error: z.string()
         })).optional().openapi({ description: "Upload failures per file" })
