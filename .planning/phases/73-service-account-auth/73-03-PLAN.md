    2. Create router: `const photosApp = new OpenAPIHono<AppEnv>()`

    3. Apply admin middleware to all routes: `photosApp.use("*", ensureAdmin)`

    4. Create GET /health route:
       - OpenAPI schema: `createRoute({ method: "get", path: "/health", ... })`
       - Handler: `typedHandler<typeof healthRoute>(async (c) => { ... })`
       - Get access token: `const token = await getPhotosAccessToken(getDb(c), c.env)`
       - Test Photos API: `fetch("https://photoslibrary.googleapis.com/v1/mediaItems:search", { method: "POST", headers: { Authorization: `Bearer ${token}` }, body: JSON.stringify({ pageSize: 1 }) })`

       - Handle 401: throw ApiError with "Authentication failed" message
       - Handle other errors: throw ApiError with error details

    5. Create placeholder GET /albums route (to be implemented in Phase 75):
       - OpenAPI schema with response 200
       - Handler returns empty array `[]` with TODO comment
       - This establishes the route for future implementation

    6. Export router: `export const photosRouter = photosApp`

    Per zero-trust security:
    - Use ensureAdmin middleware (not header-based auth)
    - No client-provided headers used for authorization decisions

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Photos/Drive routers | Admin-only access prevents unauthorized API calls |
| Workers → Google APIs | Service account credentials never exposed to client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-13 | S | Token leakage in response | mitigate | Health endpoint returns only status, not access token |
| T-73-14 | T | API call without auth | mitigate | ensureAdmin middleware on all routes |
| T-73-15 | I | Response manipulation | accept | Read-only operations, client input validated by Zod schemas |
| T-73-16 | D | D1 settings unavailable | mitigate | Retry logic in getOrRefreshToken handles transient failures |
| T-73-17 | E | Google API rate limit | accept | Health endpoint makes single call, low volume |
| T-73-18 | R | Token replay attack | accept | Service account token (not user), low value target |
</threat_model>
