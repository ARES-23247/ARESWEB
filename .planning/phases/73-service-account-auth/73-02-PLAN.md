       - `getGcalAccessToken` and `GCalConfig` from `./gcalSync.ts`
       - `ApiError` from `../shared/errors/api.ts`
       - Database query functions from middleware (getDb pattern)

    2. Create interface `TokenCache`:
       ```typescript
       interface TokenCache {
         accessToken: string;
         expiresAt: string; // ISO timestamp
       }
       ```

    3. Create `getOrRefreshToken()` helper function:
       - Parameters: `type: "drive" | "photos"`, `db: D1Database`, `env: Env`
       - Check D1 settings for cached token: `{type}_access_token`, `{type}_token_expires_at`
       - If token exists and expiresAt > Date.now() + 5 minutes (300000ms), return cached token
       - If token missing or expiring soon, call `getGcalAccessToken()` to get new token
       - Calculate new expiry: `Date.now() + 3600000` (1 hour from Google)
       - Store new token in D1 settings with upsert pattern
       - Implement retry with exponential backoff (100ms → 200ms → 400ms, 3 retries per D-07)
       - Log errors to console per D-09
       - Throw `ApiError` after retries exhausted per D-08

    4. Create `getDriveAccessToken()` function:
       - Parameters: `db: D1Database`, `env: Env`
       - Calls `getOrRefreshToken("drive", db, env)`
       - Returns access token string

    5. Create `getPhotosAccessToken()` function:
       - Parameters: `db: D1Database`, `env: Env`
       - Calls `getOrRefreshToken("photos", db, env)`
       - Returns access token string

    D1 settings key names per D-04:
    - Drive: `drive_access_token`, `drive_token_expires_at`
    - Photos: `photos_access_token`, `photos_token_expires_at`

    Lazy refresh logic per D-05:
    - Check expiry before API call
    - Refresh if within 5 minutes of expiration (buffer value)
    - No background refresh needed (lazy = on-demand)
