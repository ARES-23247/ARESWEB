  <name>Task 1: Create D1 schema for import tracking</name>
  <files>src/db/schema.ts, migrations/add_import_tables.sql</files>
  <action>
Create three new sqliteTable exports in src/db/schema.ts:

1. **imported_photos** - Tracks successfully imported photos
   - id: text().primaryKey() (UUID)
   - r2Key: text().notNull() (R2 storage key)
   - originalFilename: text().notNull()
   - googleMediaItemId: text().notNull().unique()
   - albumId: text().references(() => photo_albums.id) (nullable)
   - importedBy: text().notNull() (user email)
   - importedAt: text("imported_at").notNull() (ISO timestamp)
   - mimeType: text().notNull()
   - fileSize: text("file_size").notNull() (bytes as string)
   - Index on googleMediaItemId for deduplication check
   - Index on albumId for album queries

2. **photo_albums** - Album metadata for R2 folder mapping per D-15
   - id: text().primaryKey() (UUID)
   - googleAlbumId: text().notNull().unique()
   - name: text().notNull()
   - r2Folder: text().notNull() (sanitized folder name)
   - syncedAt: text("synced_at").notNull() (ISO timestamp)
   - mediaItemsCount: text("media_items_count") (from Google Photos API)

3. **import_audit_log** - Audit trail per D-16/D-17
   - id: text().primaryKey() (UUID)
   - mediaItemId: text().notNull() (Google Photos ID)
   - filename: text().notNull()
   - status: text().notNull() ("success" or "failed")
   - error: text() (nullable, error message if failed)
   - r2Key: text() (nullable, set on success)
   - importedBy: text().notNull() (user email)
   - importedAt: text("imported_at").notNull() (ISO timestamp)
   - Index on importedAt for time-based queries

Also create migrations/add_import_tables.sql with CREATE TABLE statements matching the schema. Use standard SQLite types (TEXT, INTEGER) with proper indexes.

Do NOT create these tables in the database yet — the migration will be applied in a later phase or via admin tool.
</action>

<task type="auto">
  <name>Task 2: Define OpenAPI contract for POST /import</name>
  <files>shared/routes/google-photos.ts</files>
  <action>
Add to shared/routes/google-photos.ts:

1. **importPhotosQuerySchema** (for request body, using z.object())
   - mediaItemIds: array(z.string()).min(1).openapi({ description: "Google Photos media item IDs to import" })
   - albumId: z.string().optional().openapi({ description: "Google Photos album ID if importing from album context" })

2. **importResultSchema** (single item result)
   - mediaItemId: z.string()
   - status: z.enum(["success", "failed"])
   - r2Key: z.string().optional() (present on success)
   - error: z.string().optional() (present on failure)
   - filename: z.string()

3. **importPhotosResponseSchema**
   - imported: z.number().openapi({ description: "Count of successfully imported photos" })
   - failed: z.number().openapi({ description: "Count of failed imports" })
   - results: z.array(importResultSchema)

4. **importPhotosRoute** (createRoute)
   - method: "post"
   - path: "/import"
   - request: { body: { content: { "application/json": { schema: importPhotosQuerySchema } } } }
   - responses: { 200: { content: { "application/json": { schema: importPhotosResponseSchema } } }, ...standardErrors }
   - tags: ["google-photos", "admin"]

Export importPhotosRoute for use in the router implementation.
</action>
